# ESP32 ECG Stream Monitor

Программа на C# для управления устройством ESP32 и регистрации биосигнала ЭКГ в реальном времени через Wi-Fi. Приложение позволяет настраивать подключение ESP32 к домашней сети, находить устройство в сети, запускать и останавливать поток данных, отображать сигнал на графике и использовать нейросетевые предсказания для разметки QRS и SPIKE.

## Возможности

- **Двухэтапное подключение ESP32**: сначала подключение к AP-режиму ESP32, затем передача параметров домашней Wi-Fi сети.
- **Стриминг сигнала по TCP**: получение данных с ESP32 в отдельном фоновом потоке.
- **Работа с ЭКГ-сигналом**: поддержка данных с датчика AD8232 или другого аналогового источника.
- **График в реальном времени**: отображение сигнала ЭКГ в основном окне приложения.
- **Нейросетевые предсказания**: возможность включить анализ сигнала моделью ONNX для определения QRS, SPIKE и QRS after spike.
- **Настройка конфигурации из интерфейса**: редактирование параметров Wi-Fi, IP, порта и данных ESP32 без ручного изменения JSON.
- **Поиск устройства в сети**: автоматическое обнаружение ESP32 в сети мобильного хот-спота Windows.
- **Кнопка открытия хот-спота Windows**: быстрый переход к настройкам мобильной точки доступа.
- **Журнал событий**: отображение статусов, ошибок и успешных операций.
- **Безопасная остановка стриминга**: перед новым подключением активный поток корректно завершается.

## Требования

- **ОС**: Windows 10/11.
- **.NET**: версия, используемая в проекте Visual Studio.
- **ESP32** с прошивкой, поддерживающей команды по TCP:
  - `PING`
  - `SET`
  - `START_STREAM`
  - `STOP_STREAM`
- **Датчик ЭКГ**: например AD8232.
- **ONNX Runtime** — если используется нейросетевая разметка.
- **OxyPlot** — для отображения графика.

## Структура проекта

Основные компоненты приложения:

- `MainForm` — главное окно программы и интерфейс пользователя.
- `StreamWorker` — отдельный поток для приема данных с ESP32.
- `NetworkManager` — общий сетевой фасад.
- `EspTcpClient` — TCP-команды для ESP32.
- `WifiProfileManager` — подключение к Wi-Fi через `netsh`.
- `NetworkScanner` — поиск устройств в сети.
- `ConfigEditorForm` — окно редактирования конфигурации.
- `EcgSegmenter` — загрузка ONNX-модели и выполнение предсказаний.

## Конфигурация

При первом запуске создается файл `config.json`.

Пример:

```json
{
  "HotspotSsid": "MyHomeWiFi",
  "HotspotPassword": "mypassword123",
  "EspDevices": [
    {
      "Name": "ESP32_ECG",
      "ApSsid": "ESP32_ECG_Streamer",
      "ApPassword": "12345678",
      "ApIp": "192.168.4.1",
      "Port": 8888,
      "HotspotIp": "192.168.137.191",
      "MacAddress": "c4:de:e2:19:2b:6c"
    }
  ]
}
```

## Установка и запуск

1. Склонируйте репозиторий:

```bash
git clone https://github.com/wabka22/esp-32-wifi
```

2. Откройте проект в Visual Studio.
3. Установите необходимые NuGet-пакеты, если они отсутствуют.
4. Соберите проект.
5. Запустите приложение.
6. При необходимости откройте настройки конфигурации в интерфейсе программы и укажите параметры ESP32.

## Общий сценарий работы

1. Включить ESP32.
2. Открыть в приложении настройки мобильного хот-спота Windows.
3. Настроить ESP32 на домашнюю сеть или сеть хот-спота.
4. Выполнить поиск устройства.
5. Запустить регистрацию сигнала.
6. При необходимости включить нейросетевые предсказания.
7. Наблюдать сигнал и события в журнале.

## Подключение AD8232 к ESP32

Пример подключения:

```text
AD8232 OUTPUT -> GPIO34
AD8232 3.3V   -> 3.3V
AD8232 GND    -> GND
```
